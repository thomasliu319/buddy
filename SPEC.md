## 1. 目标与范围

### 1.1 一句话定义
一个 macOS（Apple Silicon）桌面 Live2D 编程宠物，住在刘海下方/虚拟刘海岛位置，被动可视化 AI 编程工具（MVP 仅 Claude Code）的 token 用量与会话状态，提供编程时的情绪陪伴。

### 1.2 目标用户
使用 Apple Silicon Mac 进行 AI 辅助编程的开发者。开源分发。

### 1.3 核心价值
1. **情绪陪伴** —— 长时间独自 coding 时有"活物"在屏幕上
2. **被动信息感知** —— 不打断专注，瞄一眼就懂当前 AI 工具状态与开销
3. **设计感** —— 比同类工具（如 Codex Pets）更精致的 Live2D + 刘海岛美学

### 1.4 交互意图优先级（用户原始排序）
1. **a 被动陪伴**（最重要）：放在刘海，自己动，不打断
2. **b 状态仪表**：token / 会话信息以"瞄一眼"为目标
3. **d 可玩弄**：撸 / 喂 / 换装 / 成长（**MVP 仅"被点击有反应"，其余进 v1.1**）
4. **c 主动陪伴**：MVP 仅 2 条气泡规则；完整规则进 v1.x

## 4. 验收场景（EARS 语法）

### 4.1 主线场景

**S1｜首次启动定位**
> When 用户首次启动应用, the 系统 shall 自动检测当前主显示器是否有刘海，并将宠物定位到刘海正下方居中（无刘海则定位到状态栏正下方居中）, within 2 秒。

**S2｜数据采集与状态更新**
> When Claude Code 在 `~/.claude/projects/**/*.jsonl` 写入新事件, the 系统 shall 解析并在 5 秒内更新 token 计数与宠物状态动作。

**S3｜L1 颜色映射**
> While 本月已用预算 < 50%, the 宠物 shall 显示常态配色；达到 50% 微黄、80% 微红、100% 进入沮丧形态。

**S4｜等待输入气泡**
> When 检测到当前 Claude Code 会话「等待用户输入」超过 30 秒, the 宠物 shall 切换到 waiting 动作并弹出气泡 "我在等你哦~"。

**S5｜任务完成通知**
> When 检测到一次会话结束（jsonl 写入 stop reason）, the 宠物 shall 播放 done 动作 + 发送 macOS 原生通知。

**S6｜悬停展开 L2**
> When 鼠标悬停宠物超过 300ms, the 系统 shall 在 200ms 内展开 L2 胶囊横条；鼠标移开 1s 后收起。

**S7｜点击进 L3**
> When 用户左键单击宠物, the 系统 shall 弹出 L3 配置窗口（独立窗口，可移动）。

**S8｜拖拽与吸附**
> When 用户长按并拖动宠物超过 5px, the 系统 shall 进入拖拽模式；释放点距离刘海中心 < 100px 时 shall 自动吸附回刘海下方居中位置；否则记忆释放位置。

**S9｜静音时段**
> While 系统时间在用户配置的静音时段内（默认 22:00–09:00）, the 宠物 shall 不弹任何气泡（macOS 通知保留，可单独关）。

**S10｜性能预算**
> While 应用空闲运行, the CPU 平均占用 shall < 2%、内存常驻 shall < 200MB。

**S11｜第三方 endpoint 单价**
> When 用户在配置窗输入自定义单价（输入/输出/缓存三个数）, the 系统 shall 用此单价计算成本；未配置时 shall 仅显示 token 数、不显示 $。

**S12｜隐私底线**
> The 系统 shall 不向任何外部服务器发送 token、代码内容、会话内容、用户路径等信息。仅允许：版本检查（可关）、首次安装时的 Live2D 资源下载（一次性）。

**S13｜冷启动数据回填**
> When 应用首次启动并扫描到历史 jsonl, the 系统 shall 在后台一次性回填最近 30 天数据到本地 SQLite, within 30 秒（不阻塞 UI）。

### 4.2 边界场景

**S14｜jsonl 损坏容错**
> When 解析 jsonl 遇到非法 JSON 行, the 系统 shall 跳过该行并记录到 `~/Library/Logs/<App>/parse-errors.log`，不影响后续解析与 UI。

**S15｜Live2D 资源加载失败**
> When Live2D 模型加载失败（文件缺失/格式错误/WebGL 初始化失败）, the 系统 shall 降级为静态 PNG 占位图（同款角色 idle/coding/done 三张），所有功能保持可用，并在配置窗显示 "渲染降级" 提示。

**S16｜刘海识别失败**
> When 无法通过系统 API 判定刘海存在性, the 系统 shall 默认按"无刘海"机型处理（虚拟刘海岛模式），并允许用户在配置窗手动切换。

**S17｜首次空数据**
> When 应用首次启动且未扫描到任何 Claude Code jsonl, the 系统 shall 显示欢迎引导卡片（说明数据来源 + 隐私承诺），宠物展示 idle 动作。

**S18｜Claude Code 未安装**
> When 检测到 `~/.claude/projects/` 目录不存在, the 系统 shall 在配置窗显示引导（"未检测到 Claude Code，是否手动指定数据目录？"），并允许用户手动选择目录。

**S19｜窗口越界回收**
> When 用户的拖拽位置或多屏切换导致宠物在屏幕可见区域外, the 系统 shall 在下次启动时自动归位到默认刘海位置。

## 5. 功能行为

### 5.1 三级信息层次

| 层级 | 触发 | 信息密度 | 核心目标 |
|---|---|---|---|
| **L1** | 始终可见 | 0 数字（颜色/动作） | 不打断专注 |
| **L2** | 鼠标悬停 300ms | 3-5 个核心数字 | 瞄一眼就懂 |
| **L3** | 左键单击 | 全量数据 + 图表 | 完整配置与统计 |

### 5.2 L1 宠物本体（被动告知）

**颜色温度映射本月预算用量：**
- 0–50%：常态配色
- 50–80%：微黄
- 80–100%：微红
- ≥100%：沮丧形态

**动作集（MVP 5 个）：**
| 动作 | 触发条件 |
|---|---|
| `idle` | 无任何 AI 工具活动；眨眼、伸懒腰循环 |
| `coding` | 检测到 jsonl 新写入事件；播放敲键盘动画 |
| `waiting` | 会话「等待用户输入」状态超过 30s |
| `done` | 会话结束（stop reason 写入） |
| `sleep` | 系统空闲 > 10 分钟，或处于静音时段 |

### 5.3 L2 胶囊横条（鼠标悬停展开）

**展开形态：**
```
       ╭──────────────────────────────────────────────╮
  🐈   │  Today  2.3M ↑12%   $4.21   ●Sonnet 4.6      │
       │  Now: ai-coding · 185k tokens · 14m elapsed  │
       ╰──────────────────────────────────────────────╯
```

**字段定义：**
- `Today`：今日累计 token 总数 + 同比昨日百分比
- `$X.XX`：今日成本（按本地价目表 + 用户单价配置）
- `●Model`：当前主导模型
- `Now`：进行中的会话名 + 已用 token + 已耗时；无活动时替换为 `Last: 会话名 · 完成于 HH:MM`

**动效：** 展开 200ms ease-out；鼠标移开 1s 后收起 200ms ease-in。

**Tab 列表（MVP）：**
- **概览**：上图所示
- **会话**：最近 5 条详情（v1 改为可分页全量）
- **Token**：与概览合并（v1 拆分为多维度）
- **宠物**：宠物渲染降级开关、静音时段、刘海模式手动切换
- **设置**：Claude Code 数据目录路径、第三方 endpoint 单价、月度预算、开机自启、版本检查开关、清空数据按钮

### 5.5 刘海定位行为

**有刘海机型（MacBook Pro 14/16, Air M2/M3 15"）：**
- 默认位置：刘海正下方居中
- 刘海识别：通过 `NSScreen.safeAreaInsets.top` 判定（M 系列起 macOS 12+ 支持）
- 待机时**头部 60–80% 露出在刘海下沿**，身体下半部紧贴刘海（决策 D2）
- 工作动画时身体最多探出 80px，绝不超过状态栏 + 80px

**无刘海机型：**
- 默认位置：屏幕顶部居中、状态栏正下方
- 视觉上构造一个"虚拟刘海岛"容器，行为完全一致

**多显示器：** 跟随主显示器（系统设置 → 显示器 → 主显示器）

### 5.6 气泡规则（MVP 仅 2 条）

| 规则 | 触发 | 文案 | 时机 |
|---|---|---|---|
| R1 | 会话「等待输入」超过 30s | "我在等你哦~" | 同时切换 waiting 动作 |
| R2 | 一次会话结束 | "搞定啦！🎉"（动态：根据用时长短换文案） | 同时播放 done 动作 + macOS 通知 |

**静音时段（默认 22:00–09:00）内仅抑制气泡，macOS 通知按系统设置。**

### 5.7 数据采集

**数据源（MVP）：** 仅 `~/.claude/projects/**/*.jsonl`

**采集字段（每条事件）：**
- 时间戳、工具来源、模型、输入 token、输出 token、缓存读、缓存写、计算成本、项目路径、会话 ID、是否第三方 endpoint

**存储：** 本地 SQLite，路径 `~/Library/Application Support/<App>/data.db`

**回填：** 首次启动扫描最近 30 天 jsonl；之后增量监听文件变更

## 6. 技术约束

### 6.1 性能
- 空闲 CPU 平均 < 2%
- 常驻内存 < 200MB
- 启动时间 < 2 秒（冷启动）
- 数据回填 < 30 秒（不阻塞 UI）

### 6.2 隐私
- 所有数据**仅本地**，不上传任何外部服务器
- 仅允许两类外发：版本检查（可关）、首次 Live2D 资源下载（一次性）
- 配置窗提供"清空所有数据"按钮（不可恢复）
- 不收集匿名遥测

### 6.3 平台
- 最低系统：macOS 13 Ventura
- 仅 Apple Silicon（M1+）
- 不支持 Intel Mac
- 不支持 Mac App Store 沙箱（用户直接 dmg 安装）

### 6.4 资产授权
- **MVP 默认模型：Mao**（Live2D Cubism 官方 Sample，~15 motion，"宠物"气质最强）— 决策 D5
  - 路径：`public/assets/live2d/mao/`
  - 授权：Live2D Sample Data License（仅可用于 SDK 学习/演示）
  - **v2 上 Mac App Store 前必须替换**为：自创模型 / CC0 模型 / 商业授权（已记录 v2 路线图） — 决策 D3
- 不内嵌付费资产
- 所有第三方资产授权写入 `LICENSE-3RD-PARTY.md`

### 6.5 分发
- GitHub Release 发 .dmg
- 必须代码签名 + 公证（避免 Gatekeeper 警告）；如无开发者账号则 v0 接受 "右键打开" 流程

### 6.6 技术栈决策（v1.1 — 人工检查点 ② 已确认 2026-05-05）

**苹果设计风执行计划：**
- 字体：`-apple-system, BlinkMacSystemFont, "SF Pro Display"`；数字 `tabular-nums`
- 毛玻璃：Tauri `vibrancy: "fullscreen-ui"`（系统级）+ CSS `backdrop-filter` 兜底

| Tailwind 4 较新 | fallback 3.4 |
| 单价计算浮点误差 | Rust 后端用 `rust_decimal` |
| Live2D 资源版权 | MVP 用 Cubism Sample (Mao)，v2 上架前替换；授权写入 LICENSE-3RD-PARTY.md |
